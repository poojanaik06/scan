# CertiScan

CertiScan scans a course certificate QR code, authenticates that it points to the
issuer's canonical certificate page, validates its private token, and reads the
issuer registry to report whether the course is completed, incomplete, revoked,
or unknown.

## Run

```bash
npm install
npm test
npm run dev
```


The demo includes verified, incomplete, and revoked records. Click **Open sample
QR**, save or display the QR, and scan it with the camera or image uploader.

## Verification model

- QR codes generated by the issuer point to
  `/issuer/certificates/:certificateId?token=...`.
- CertiScan accepts QR URLs only from its own configured issuer origin.
- The token must match the issuer registry record.
- Completion is decided from the current registry status, not text embedded in
  the QR code.
- Every result links to the canonical issuer page.

This demo represents one issuer. Supporting Coursera, Udemy, university portals,
or other websites requires an approved API or a separate issuer adapter for each
site; arbitrary web pages cannot be verified reliably or safely with one generic
scraper.
